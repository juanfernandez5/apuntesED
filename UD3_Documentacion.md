# Documentación

La documentación es una parte fundamental del desarrollo de software. En este módulo aprenderás a documentar tus proyectos utilizando dos herramientas esenciales:

- **Markdown**: Para documentar el proyecto completo (README, guías de usuario, documentación general). Es el lenguaje de marcado que usarás en GitHub y en archivos `.md`.
- **Javadoc**: Para documentar el código fuente de Java (clases, métodos, atributos). Genera documentación HTML profesional automáticamente.

Ambas herramientas son estándares de la industria y te permitirán crear documentación clara, profesional y fácil de mantener.

## Diferencia entre Markdown y Javadoc

| Aspecto | Markdown | Javadoc |
|---------|----------|---------|
| **Propósito** | Documentar el proyecto | Documentar el código fuente |
| **Archivos** | README.md, guías, wikis | Archivos .java |
| **Ubicación** | Raíz del proyecto, docs/ | Dentro del código fuente |
| **Salida** | Archivos .md (GitHub los renderiza) | HTML generado automáticamente |
| **Audiencia** | Usuarios, desarrolladores, público | Desarrolladores |
| **Contenido** | Descripción, instalación, uso | API, métodos, parámetros |

## Cuándo usar cada uno

### Usa Markdown para:
- ✅ README del proyecto
- ✅ Guías de instalación y uso
- ✅ Documentación de arquitectura
- ✅ Notas de versión (CHANGELOG)
- ✅ Contribuir al proyecto (CONTRIBUTING.md)

### Usa Javadoc para:
- ✅ Documentar clases y sus responsabilidades
- ✅ Explicar qué hace cada método
- ✅ Describir parámetros y valores de retorno
- ✅ Documentar excepciones que se lanzan
- ✅ Generar API reference automática
